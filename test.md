## Problem Statement

Procurement managers at mid-market and enterprise companies negotiate the same vendor renewals year after year, but institutional memory about what tactics worked â€” and when â€” is fragmented across Slack threads, call notes, email archives, and the heads of people who left the company. They walk into every renewal with a generic playbook and forget what moved the needle last time. LEVERAGE is a pre-call briefing tool that turns Hindsight Cloud memory into scored, evidence-backed negotiation tactics and updates those scores live as new calls are logged.

For the hackathon, the deliverable is a working demo of the procurement wedge: one dashboard, three seeded vendors (NexaCloud, DataPipe, SecureNet), a briefing screen with a before/after memory toggle as the hero moment, and a post-call logger that visibly updates tactic confidence scores.

## What Makes This Cool

- **Memory IS the product.** Without Hindsight, the app is a ChatGPT wrapper. With it, each vendor accumulates a private, compounding knowledge base that gets sharper every call. The demo makes this tangible in one toggle click.
- **Evidence-scored tactics.** Every recommendation comes with a confidence score (0.0â€“1.0), a track record (e.g. "2 for 2"), timing guidance, and the actual history behind it. Feels less like an LLM guess, more like a coach who's watched every past call.
- **Live score evolution.** After logging a call, confidence scores diff in real time. A successful tactic ticks up. A failed one ticks down. The memory visibly learns in front of the judge.
- **Hackathon-to-company arc.** The narrative expands cleanly from "vendor renewals" to "every repeated B2B negotiation" â€” sales close, legal settlements, M&A, hiring packages. Judges and investors see a feature and a company in the same demo.

### Confidence score formula

Scores must be stable across renders â€” an LLM picking floats will drift. Formula lives in `briefing.py`:

```python
def confidence(successes: int, total_uses: int, months_since_last_use: float) -> float:
    if total_uses == 0:
        return 0.0
    base = successes / total_uses
    recency_decay = max(0.3, 0.9 ** months_since_last_use)
    return round(base * recency_decay, 2)
```

The `BRIEFING_PROMPT` asks the LLM to emit `successes`, `total_uses`, and `last_used_date` for each tactic. Backend computes `confidence` deterministically before returning. LLM never picks the float directly â€” it just extracts structured facts from memory.
